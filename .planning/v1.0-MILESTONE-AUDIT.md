## Integration Issues

### High — XAddr Host Address (DEV-02, DEV-03, DISC-01)

`OnvifServer::run()` constructs all service XAddrs as `http://0.0.0.0:<port>/onvif/...`. GetCapabilities, GetServices, and WS-Discovery ProbeMatch all broadcast these unroutable addresses. Real ONVIF clients (Frigate, ODM, Home Assistant) cannot connect unless on localhost.

**Fix:** Add `advertised_host(host: &str)` to `OnvifServerBuilder`. Use it in XAddr construction. Default to `0.0.0.0` for backward compat but document that consumers must set this to their LAN IP.

### Medium — Auth Test Gap (DEV-05, INFRA-07)

Both WS-Security credential tests are `#[ignore]` with `todo!()` bodies. The ODM smoke test bypasses auth by calling handlers directly. No test exercises the credential rejection path over HTTP.

**Fix:** Write HTTP-level auth tests using a real TCP listener (like `device_server_binds_and_serves_auth_exempt_op` already does).

### Low — PTZ Constructor Consistency (PTZ-01)

`PTZServiceHandler` is constructed via struct literal in server.rs rather than `::new()`, unlike all other handlers. Fragile on future field additions.

**Fix:** Change `PTZServiceHandler { svc: ptz_svc }` to `PTZServiceHandler::new(ptz_svc)`.
