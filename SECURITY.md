# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities by email to **jhogendorn@gmail.com**. Include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a minimal proof of concept.
- Any relevant version or environment information.

You will receive an acknowledgement within 72 hours. We aim to triage and confirm
reports within 7 days and to release a fix within 30 days for confirmed
vulnerabilities, depending on severity and complexity.

We will credit reporters in the release notes unless you prefer to remain anonymous.
